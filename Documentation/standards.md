## 2. Coding Standards

### 2.1 Naming Conventions
- Functions: snake_case → add_record(), generate_report()
- Variables: snake_case → item_id, item_name, inventory_status
- Constants: ALL CAPS → MAX_RECORDS = 25
- Files: lowercase with underscores → inventory_stubs.py

### 2.2 Indentation and Comments
- Use 4 spaces per indentation level. No tabs.
- Every function must include a docstring.
- Use inline # comments to explain logic.
- Mark pending code with: # TODO: Implementation in Phase 3

### 2.3 Status Values
- "In Stock"
- "Low Stock"
- "Out of Stock"

### 2.4 Data Fields
- Item ID – Unique identifier for each item
- Item Name – Name or description of the item
- Inventory Status – Current stock level

## 3. Git Workflow

### 3.1 Commit Message Format
[Action] [What was done] – [Your Name]

Examples:
- Added standards.md to Documentation folder – Ronny
- Added add_record() stub with docstring – Melissa
- Added generate_report() stub – Antoine

### 3.2 Commit Rules
- Each member must make at least one commit and push per phase
- Always pull latest changes before pushing
- Never commit without # TODO notes where needed
